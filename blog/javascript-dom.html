HTML, CSS, & JavaScript

An anology that I like to use to describe the relationship between HTML, CSS, and Javascript, is to think of a webpage as a coffee shop. The HTML of a web page can be compared to the base structure and architecture of the coffee shop, the walls, floors, windows, roof. The CSS is the outer style and interior design, such as the funiture, wall paper, carpet, wall decor, and the painting on the outside. With both the HTML & CSS, we have a nice looking coffee shop, but a coffee shop needs more to it in order to operate as a proper establishment. The JavaScript in this instance, covers all the inner working parts that help the coffee shop function, such as the barristas to take and serve orders, the coffee machines to make the orders, the register to keep track of sales and profits, the clocking system to record workers hours, the inventory management system to keep track of and make new orders, etc..

Loops & Control Flow

In JavaScript, loops are a method to perform functions repeatedly based on the type of loop it is (i.e. for, while, do/while). Control flow is the order that the computer reads the code in to determine how it will perform functions, by default it will be read from first line to last, however, there are keywords that can be used in order to change the structure that the computer reads in. However, let's then say that this personal also wants to set a limit to how many cups of coffee they drink per day, and want to remind themselves to not go over the limit. This is when control flow comes it to add a break to our structure once that limit has been hit.

With this addition, the previous loop of drink coffee then add to journal, will function as long as the daily total is less than the persons daily limit. However, if the persons daily total is equal to their daily limit, then the control flow will shift to the next part of the loop, where they will document in their journal a remainder that their daily limit has been reached, and will stop drinking coffee for the day.

The DOM, much like the apple tree allows us to visualize, interact with, and modify each of the objects that it contains.
